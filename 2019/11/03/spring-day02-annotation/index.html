spring_day02_ioc

1.使用注解来管理spring ioc

用于创建对象

1.@Component 作用：把资源让spring管理，相当于在xml中配置了一个bean 属性：指定bean的id，如果不指定value的属性，默认bean的id为当前类名，首字母小写
2.@Controller 一般用于表现层的注解
3.@Service 一般用于业务层的注解
4.@Respository一般用于持久层的注解(如果注解中只有一个属性要赋值时，且名称为value，value在赋值时可以不写)

用于注入数据

1.@Autowired 作用：自动按照类型注入，当使用注解注入属性时，set方法可以省略，他只能注入其他bean类型当多个类型匹配时，要使用对象的变量名称作为bean的id
2.@Qualifier作用自动注入的基础上，在按照Bean的id注入,给字段注入时不能单独使用，必须和@Autowire一起使用。属性：指定bean的id
3.@Resource作用:按照bean的id注入，只能注入其他bean类型。属性:指定bean的id
4.@Value作用：注入基本数据类型和String类型value:指定值

改变作用范围的

1.@Scope 作用：指定bean的作用范围。属性：value:取值singleton prototype request session globalsession
2.@PostConstruct初始化方法
3.@PreDestroy销毁方法

注解配置类

1.@Configuration作用指定当前是一个配置类获取容器时需要使用AnnotationApplicationContext value指定配置类的字节码
2.@ComponentScan用于在spring在初始时要扫描的包
3.@Bean 该注解只能写在方法上，使用这个方法创建一个对象，放入spring容器中 name：指定id


4.@PropertySource作用：加载。properties 文件中位置,value[]指定文件位置
5.如果有两个配置类但是要建立配置文件与配置文件之间的关系用@Import ,导入其他配置类，在引入其他配置类时可以不用写@Comfiguration

6.没有配置文件如何获取容器呢，也可以通过注解，ApplicationContext ac = new AnnotationConfigApplicationContext(SpringConfiguration.calss)
