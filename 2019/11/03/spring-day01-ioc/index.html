##关于jdbc的简单用法
1.注册驱动
DriverManager.registerDriver(new com.mysql.jdbc.Driver());
或者Class.forName(“com.sql.jdbc.Driver”);
2.获取连接
Connection conn = DriverManager.getConnection(“jdbc:mysql://localhost:3306/数据库名称，”root”,”admin”);
3.获取操作数据库的预处理对象
PrepareStatement pstm = conn.prepareStatement(“select * from account”);
4.执行sql语句得到结果集
ResultSet rs = pstm.executeQuery();
5.便利结果集
while(rs.next()){System.out.println(rs.getString(“name”);)}
6.释放资源 rs.close() pstm.close(); conn.close(); 规则先打开的最后关闭

##spring中使用xml进行配置
#1.spring中两个核心功能一个是ioc一个是aop,ioc是一个容器，使用ioc容器可以很好的降低程序间的耦合，两个特性，控制反转和依赖注入
将对象的实例化交给spring去创建减少使用new的关键字，通过注入的方式给变量对象赋值
一般进行xml进行配置的时候xml中主要使用标签，其中含有class属性和id属性，class一般是所创建对象的包名的全部路径，id是实例化对象的名字，通过这
两个属性来确定具体的位置。init-method：指定类中的初始化方法名称。 destroy-method：指定类中销毁方法名称。scope：指定对象的作用范围

#2.在主函数中如何使用java bean
1.获取核心容器的对象
ApplicationContext applicationContext = new ClassPathXmlApplicationContext(“bean.xml”);
2.根据id获取Bean对象
IAccountService accountService = (IAccountService)accountService.getBean(“配置文件中的id”)；
3.根据创建好的对象调用它的方法
accountService.saveAccount();
#3.在bean.xml中的DI依赖注入
1.复杂类型/集合类型注入的方式：有三种第一种：使用构造函数提供第二种：使用set方法提供第三种：使用注解提供
2.构造函数注入：使用的标签:constructor-arg 标签出现的位置：bean标签的内部标签中的属性 type：用于指定要注入的数据的数据类型，该数据类型也是构造函数中某个或某些参数的类型 index：用于指定要注入的数据给构造函数中指定索引位置的参数赋值。
索引的位置是从0开始 name：用于指定给构造函数中指定名称的参数赋值(使用较少)
<constructor-arg value = "" name = ""> <constructor-arg name = "复杂属性" ref="值" > 3.配置一个日期对象
4.使用set方法然后通过配置注入, 复杂类型的注入/集合类型的注入用于给List结构集合注入的标签： list array set 用于个Map结构集合注入的标签:map props 结构相同，标签可以互换 –>
<property name = "myStrs"> <set><value>AAA</value></set></property><property name = “myList” AAA
<property name = "myMap"> <props><prop key = "test">ccc</prop></props> 

  简单类型：`<bean id = "" class ="">  <property name = "name" value ="text"> </property>
